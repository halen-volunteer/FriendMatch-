## 一、项目总体概述

`FriendMatch` 是一个面向社交与团队协作场景的即时通信平台，核心目标是围绕“用户连接、团队协作、聊天沟通、平台治理、安全管理、搜索推荐”构建完整业务闭环。

当前项目可以分为两大使用侧：

- `用户端`：普通用户使用的注册、登录、聊天、好友、团队、搜索、通知、设备、反馈等功能
- `管理端`：平台管理员使用的举报审核、申诉处理、处罚执行等治理功能

系统整体模块主线可以概括为：

```text
认证登录
  → 用户关系与资料管理
  → 团队与成员协作
  → 私聊 / 群聊 / 消息增强
  → 搜索与推荐
  → 平台治理与安全
```

## 二、用户端模块与功能说明

## 1. 用户认证模块

### 1.1 登录
用户可以通过登录页进入系统。

支持功能：
- 账号或邮箱登录
- 密码校验
- 图形验证码校验
- 登录后自动获取当前用户信息
- 根据身份自动跳转普通用户端或管理端

### 1.2 注册
用户可以注册新账号。

支持功能：
- 输入账号
- 输入邮箱
- 获取邮箱验证码
- 设置密码与确认密码
- 注册成功后跳转登录页

### 1.3 忘记密码
用户可以通过邮箱验证码找回密码。

支持功能：
- 输入注册邮箱
- 获取邮箱验证码
- 重置新密码
- 校验两次密码一致性
- 成功后跳转登录页

## 2. 会话与聊天模块

这是当前项目的核心使用模块之一，已经实现微信桌面版风格的三栏式布局。

### 2.1 会话首页
会话页是用户进入系统后的默认主页面。

支持功能：
- 左侧图标导航栏常驻
- 中间会话列表常驻
- 右侧显示当前聊天内容或空状态页
- 会话搜索
- 会话未读数量显示
- 当前会话高亮

### 2.2 私聊功能
用户可以和好友进行一对一聊天。

支持功能：
- 进入私聊会话
- 发送文本消息
- 上传图片或文件消息
- 消息撤回
- 消息举报
- 消息收藏
- 消息置顶
- 消息搜索
- 清理发送失败消息
- 本地消息缓存
- 已读状态更新
- WebSocket 实时接收新消息

### 2.3 群聊功能
用户可以在团队会话中进行群组聊天。

支持功能：
- 进入团队群聊
- 群消息实时收发
- 图片 / 文件消息发送
- 群公告查看与编辑
- 消息撤回 / 收藏 / 举报 / 置顶 / 搜索
- 清理发送失败消息
- 已读更新
- 群公告实时更新通知

### 2.4 消息增强能力
聊天系统除基础收发外，还支持增强功能。

支持功能：
- 图片消息
- 文件消息
- 消息收藏
- 消息置顶
- 会话内搜索
- 消息举报
- 失败消息重试与删除
- 本地数据库缓存
- 文件上传能力

## 3. 用户资料与关系模块

### 3.1 我的资料
用户可以维护自己的个人信息。

支持功能：
- 查看个人资料
- 编辑昵称、头像、简介、标签等信息
- 进入个人中心页

### 3.2 查看他人资料
用户可以查看其他用户主页。

支持功能：
- 查看对方头像、昵称、账号、简介、标签
- 查看对方在线状态
- 发起好友申请
- 直接进入私聊
- 将对方加入黑名单

### 3.3 好友管理
用户可以维护自己的好友关系。

支持功能：
- 查看好友列表
- 查看好友申请列表
- 同意好友申请
- 拒绝好友申请
- 删除好友
- 从好友列表发起聊天

### 3.4 黑名单管理
用户可以管理被拉黑用户。

支持功能：
- 查看黑名单列表
- 解除拉黑

---

## 4. 团队管理模块

团队模块承担协作型社交能力。

### 4.1 团队广场
用户可以浏览和搜索团队。

支持功能：
- 查看团队列表
- 搜索团队
- 创建团队
- 公开团队申请加入
- 密码加入团队
- 查看团队详情

### 4.2 团队详情
用户可以查看团队信息与执行团队操作。

支持功能：
- 查看团队名称、简介、标签、人数
- 查看自己在团队中的角色
- 进入团队群聊
- 查看成员列表
- 编辑团队资料（队长）
- 退出团队
- 解散团队（队长）
- 审核入队申请（队长 / 管理员）

### 4.3 成员管理
团队内支持角色与成员管理。

支持功能：
- 查看团队成员列表
- 查看成员角色
- 查看成员是否被禁言
- 移除成员
- 禁言 / 解禁成员
- 设置管理员
- 降级成员角色
- 转让队长
- 开启 / 关闭全员禁言

## 5. 搜索与推荐模块

### 5.1 搜索中心
用户可以搜索用户与团队。

支持功能：
- 用户搜索
- 团队搜索
- 搜索历史展示

系统根据已有策略给出推荐结果。

- 推荐用户列表

- 查看推荐对象详情
- 记录推荐点击行为

## 6. 通知与系统消息模块

### 6.1 通知中心
用户可以查看平台通知。

支持功能：
- 查看通知列表
- 按通知类型展示不同内容

- 好友申请
- 好友拒绝
- 入群通过 / 拒绝
- 被移出团队
- 账号异常
- 处罚通知
- 反馈回复

支持功能：

支持功能：

- 查看处罚开始与结束时间

### 8.1 反馈中心
用户可以向平台提交意见与问题。

支持功能：
- 提交功能问题反馈
- 提交违规举报类反馈
- 提交处罚申诉类反馈

- 填写补充说明

### 8.3 申诉中心
用户可以对治理结果发起申诉。

支持功能：
- 填写关联举报 ID
- 选择举报类型
- 提交申诉原因
- 查看自己的申诉记录
- 查看申诉处理状态

## 三、管理端模块与功能说明

管理端用于平台治理、安全审核和处罚处理，目前已具备完整的基础治理闭环。

## 1. 管理端总览模块

### 1.1 控制台首页
管理员进入后台后看到治理总览。

支持功能：
- 查看待处理用户举报数量
- 查看待处理团队举报数量
- 查看待处理申诉数量
- 查看整体治理入口

---

## 2. 用户举报管理模块

管理员可以集中处理用户举报。

支持功能：
- 查看用户举报列表
- 根据状态筛选举报
- 查看举报原因与说明
- 确认违规
- 驳回举报

---

## 3. 团队举报管理模块

管理员可以处理团队违规举报。

支持功能：
- 查看团队举报列表
- 查看举报详情
- 确认处理
- 驳回举报

---

## 4. 申诉处理模块

管理员可以审核用户申诉。

支持功能：
- 查看申诉列表
- 查看关联举报
- 查看申诉理由
- 通过申诉
- 驳回申诉

---

## 5. 处罚中心模块

管理员可以执行和撤销处罚。

支持功能：
- 输入目标用户并执行处罚
- 记录处罚原因
- 查询指定用户处罚记录
- 查看累计违规次数
- 撤销处罚

该模块是平台治理链路的重要一环，通常与举报和申诉联动使用。

## 四、公共支撑模块

除了用户端与管理端的页面功能外，项目还具备一组底层支撑能力。

## 1. WebSocket 实时通信
支持聊天与公告的实时推送：
- 私聊消息实时推送
- 群聊消息实时推送
- 消息撤回广播
- 群公告更新广播
- 系统通知广播

## 2. 本地消息缓存
支持聊天消息的本地持久化：
- 会话消息保存到本地数据库
- 撤回状态本地同步
- 失败消息清理与恢复

## 3. 文件上传能力
支持聊天中的文件与图片发送：
- 图片上传
- 文件上传
- 上传进度反馈
- 大文件上传支持

## 4. 路由与权限控制
系统对普通用户和管理员进行分流：
- 未登录用户访问受限页面时跳转登录
- 管理端页面需要管理员身份
- 已登录用户访问认证页时自动重定向
