# FriendMatch 高校兴趣社群聊天系统

基于 Spring Boot + Vue 3 + WebSocket 构建的高校兴趣社群聊天系统，项目定位为个人课堂学习与毕业设计实践项目。系统围绕“兴趣匹配 + 社群协作 + 实时聊天 + 平台治理”展开，实现了单聊/群聊实时通信、离线消息补推、团队管理、用户关系管理、搜索推荐、内容风控与平台治理等完整能力，兼顾用户交互体验与后端工程实践。

## 项目亮点

- 实时通信链路：基于 WebSocket 实现单聊、群聊与系统通知推送，支持离线消息补推、未读数统计、在线状态维护与消息已读回执。
- 会话状态与消息治理：支持消息撤回、编辑、收藏、置顶、消息搜索等能力，增强聊天场景下的消息可控性与交互完整性。
- 社群协作能力：围绕建队、入队审批、成员管理、角色分级、队长转让、全员禁言与成员禁言等场景实现完整业务规则。
- 用户关系与隐私控制：支持好友申请、好友管理、黑名单管理、隐私设置等功能，提升陌生社交场景下的关系安全性。
- 搜索推荐与缓存优化：支持用户搜索、团队搜索、搜索历史、热门搜索与轻量级推荐，并结合 Redis 缓存优化高频访问场景。
- 平台治理与内容安全：实现举报、处罚、申诉、反馈、系统通知等治理链路，并接入 AI 内容审核能力辅助识别违规文本。
- 文件上传体验优化：基于七牛云对象存储支持前端直传，前端实现大文件分片上传、断点续传与上传进度反馈。
- 工程化实践：采用前后端分离架构，具备统一异常处理、参数校验、权限拦截、缓存设计与模块分层能力。

## 核心功能模块

### 1. 用户认证与用户管理
- 用户注册、登录、忘记密码
- 邮箱验证码、图形验证码、Token 鉴权
- 用户资料维护、头像上传、兴趣标签设置
- 隐私设置、邮箱搜索权限控制
- 在线状态维护、设备管理、登录日志记录

### 2. 好友关系与隐私控制
- 好友申请、同意、拒绝、删除
- 黑名单添加与移除
- 资料可见范围与消息接收范围控制
- 基于关系链路的权限判断与隔离

### 3. 团队（兴趣社群）管理
- 团队创建、编辑、解散
- 入队申请、审批、邀请、退队
- 成员列表管理、角色调整、队长转让
- 全员禁言、成员禁言、团队公告

### 4. 实时聊天系统
- 单聊与群聊消息实时收发
- 历史消息查询
- 离线消息补推
- 未读数统计、消息已读回执
- 消息撤回、编辑、收藏、置顶、搜索

### 5. 平台治理与内容风控
- 用户举报、团队举报、消息举报
- 处罚执行、处罚撤销、处罚记录查询
- 用户申诉、反馈处理
- 系统通知实时推送
- AI 文本审核与敏感词过滤辅助治理

### 6. 搜索与推荐
- 用户搜索、团队搜索
- 搜索历史、热门搜索
- 基于标签、关系、活跃度等信息的轻量级推荐
- 推荐结果缓存与定时刷新

### 7. 文件上传
- 七牛云上传凭证下发
- 前端直传图片与文件
- 大文件分片上传
- 断点续传与上传进度反馈

## 本地运行

### 环境准备
- JDK 8+
- MySQL 8+
- Redis 6+
- Node.js 16+
- Maven 3.6+

### 1. 后端启动

进入项目根目录后，导入数据库脚本：

- 执行 `SQL/friendmatch.sql`

准备配置文件：

- 将 `src/main/resources/application.example.yaml` 复制为 `src/main/resources/application.yaml`
- 按本地环境修改 MySQL、Redis、邮箱、七牛云等配置项

启动项目：

- 运行 `src/main/java/com/zero/usercenter/UserCenterApplication.java`

默认端口：

- 后端：`http://localhost:8081`

### 2. 前端启动

进入前端目录：

```bash
cd VueFormatCode
```

安装依赖：

```bash
npm install
```

启动项目：

```bash
npm run dev
```

默认端口：

- 前端：`http://localhost:5173`
