# FriendMatch 项目测试方案（终版草案）

## 1. 目的与范围

本文档用于指导 `VueFormatCode` 前端与 `FriendMatch` 后端的功能测试、联调测试、异常测试、回归测试、基础性能测试与上线验收。  

覆盖模块：

- 登录认证
- 用户资料与隐私
- 好友关系
- 黑名单
- 团队
- 聊天
- OSS 上传
- 搜索与推荐
- 系统通知
- 设备管理
- 在线状态
- 举报
- 申诉
- 反馈
- 处罚

覆盖接口总数：105。

> 说明：完整性不等于“接口都列出来了”。完整测试方案至少要覆盖：接口、异常、权限、联调、回归、性能、安全。

## 6. 通用检查清单

### 6.1 所有接口都要检查

- 方法、URL、参数、Header 是否正确
- `Authorization` 是否按预期携带
- 响应结构是否符合项目约定：`success/message|errorMsg/data/total`

### 6.2 关键接口必须补测异常

- 未登录访问
- token 无效/过期
- 缺少必填参数
- 参数类型错误
- 资源不存在
- 权限不足
- 重复提交

### 6.3 页面联调用例必须补查

- 按钮/表单能否触发
- 成功/失败提示是否清晰
- 列表是否刷新
- 页面状态是否同步
- 路由跳转是否正确

## 7. 模块测试策略

### 7.1 登录认证

关键链路：验证码 -> 登录 -> 获取当前用户；注册 -> 邮箱验证码 -> 注册；忘记密码 -> 新密码登录。  
必测异常：错误密码、错误验证码、邮箱已注册、验证码过期、未登录访问 `/api/auth/me`。  

### 7.2 用户资料与隐私

关键链路：修改头像/昵称/简介 -> 重新获取资料 -> 页面同步显示；修改隐私设置 -> 其他账号访问验证。  
必测异常：昵称超长、tags 非法、访问不存在用户、未登录修改资料。  
联调重点：头像、个人页、聊天页显示一致。

### 7.3 好友关系

关键链路：发起申请 -> 对方查看申请 -> 同意/拒绝 -> 列表更新 -> 删除好友。  
必测异常：不能加自己、重复申请、已是好友重复申请、非接收方审批。

### 7.4 黑名单

关键链路：拉黑 -> 验证好友/私聊受影响 -> 解除拉黑 -> 能力恢复。  
必测异常：重复拉黑、解除未拉黑对象、拉黑不存在用户。

### 7.5 团队

关键链路：创建团队 -> 申请/密码加入 -> 审批 -> 成员列表 -> 修改角色 -> 转让队长 -> 全员禁言/成员禁言 -> 退出/解散。  
必测异常：密码错误、满员加入、普通成员审批失败、普通成员禁言失败、已解散团队继续操作。  
权限重点：普通成员、管理员、队长的操作边界。

### 7.6 聊天

关键链路：

- 私聊：发送 -> 历史 -> 编辑 -> 撤回 -> 收藏 -> 置顶 -> 搜索 -> 已读
- 群聊：发送 -> 历史 -> 公告 -> 兼容禁言接口
- 举报：举报 -> 查询状态 -> 管理员查看上下文 -> 管理员处理 -> 发起申诉
必测异常：空消息、非发送者编辑/撤回、非成员发群消息、举报不存在消息、非管理员访问管理接口、已禁言成员仍可发言。  
联调重点：聊天页消息状态、本地列表同步、举报弹窗闭环、群公告展示。

### 7.7 OSS 上传

关键链路：获取预签名 -> 实际上传 -> 返回 URL 可访问 -> 头像/消息中可用。  
必测异常：文件为空、类型非法、文件过大、token 过期。

### 7.8 搜索与推荐

关键链路：搜索 -> 搜索历史 -> 热搜 -> 联想 -> 推荐列表 -> 点击推荐上报。  
必测异常：空关键词、超长关键词、非法 `type/searchType`、推荐点击非法参数。  
说明：推荐点击接口测试应按当前真实实现字段执行。

### 7.9 系统通知

关键链路：业务动作触发通知 -> 未读数变化 -> 列表查询 -> 标已读 -> 删除。  
联调重点：举报/反馈/处罚处理后通知是否生成。

### 7.10 设备管理

关键链路：绑定设备 -> 查询列表 -> 信任设备 -> 下线 -> 删除设备。  
必测异常：重复绑定、删除不存在设备、操作非本人设备。

### 7.11 在线状态

关键链路：设置状态 -> 心跳 -> 查询状态 -> 主动下线。  
联调重点：聊天页/好友页在线状态展示一致。

### 7.12 举报 / 申诉 / 反馈 / 处罚

要求覆盖完整业务闭环：提交 -> 查询/列表 -> 管理员处理 -> 用户查看结果/恢复状态。
必测异常：越权处理、重复处理、资源不存在。
补充要求：申诉提交后应随机分配给“未处理过该举报”的管理员；管理员待处理列表只展示分配给自己的申诉；非分配管理员处理应失败。

## 9. 105 个接口最低执行要求

> 每个接口至少执行 1 条正常流；关键接口再补异常和权限用例。

### 9.1 LoginController（9条）

- `TC-001-001` `GET /api/auth/captcha`：返回图片流与 `captchaId`
- `TC-001-002` `POST /api/auth/login`：正确登录成功
- `TC-001-003` `POST /api/auth/login`：错误密码失败
- `TC-001-004` `POST /api/auth/login`：错误验证码失败
- `TC-001-005` `POST /api/auth/register`：注册成功
- `TC-001-006` `POST /api/auth/register`：重复邮箱失败
- `TC-001-007` `POST /api/auth/code`：发送邮箱验证码成功
- `TC-001-008` `POST /api/auth/me`：获取当前用户成功
- `TC-001-009` `POST /api/auth/forget`：忘记密码成功

### 9.2 其他控制器最低执行要求

- UserManagementController：资料更新、隐私读取、隐私更新、指定用户资料、用户列表、用户搜索
- FriendManagementController：发起申请、好友列表、同意、拒绝、删除、申请列表
- BlacklistManagementController：拉黑、解除拉黑、黑名单列表
- TeamController：创建、列表、搜索、详情、更新、解散、申请加入、密码加入、邀请、退出、待审批、审批、成员列表、按角色筛选、移除成员、角色更新、兼容角色接口、转让队长、兼容转让接口、全员禁言、成员禁言、成员解除禁言
- ChatController：私聊发送、私聊历史、群聊发送、群聊历史、编辑、撤回、收藏、取消收藏、收藏列表、置顶、取消置顶、置顶列表、搜索、消息举报、举报状态、管理员上下文、管理员列表、管理员处理、举报申诉、已读、未读数、设置公告、获取公告、兼容全员禁言、兼容成员禁言、兼容解除禁言、已取消群名接口返回失败提示
- OssController：预签名获取成功且可用于上传
- SearchRecommendController：搜索用户、搜索团队、搜索历史、清空历史、热搜、联想、推荐用户、推荐团队、推荐点击
- SystemNoticeController：未读数、通知列表、标已读、删除通知
- DeviceController：绑定设备、设备列表、信任、下线、删除
- OnlineStatusController：设置状态、心跳、查询状态、主动下线

## 10. 高风险异常场景清单

### 10.1 鉴权类

- 未登录访问受保护接口
- token 无效/过期
- 使用他人 token 访问本人资源

### 10.2 参数类

- 缺少必填参数
- 参数类型错误
- 枚举值非法
- 页码为 0 / 负数 / 超大值
- 内容为空 / 仅空格
- 文本超长

### 10.3 业务状态类

- 重复加好友
- 重复申请入队
- 已撤回消息再次撤回
- 已处理举报再次处理
- 已处理申诉再次处理
- 已解散团队继续操作

### 10.4 权限类

- 普通成员审批入队
- 普通成员修改角色
- 普通用户处理举报/申诉/反馈
- 非管理员执行处罚
- 非本人删除设备

### 10.5 上传类

- 非法文件类型
- 超大文件
- token 过期上传
- 上传成功但文件不可访问

## 35. 测试环境搭建指南

### 35.1 后端环境准备

1. 安装 JDK 21
2. 安装 MySQL 8.x，并创建数据库 `friendmatch`
3. 安装 Redis，并确保 `127.0.0.1:6379` 可访问
4. 安装 RabbitMQ，并确保 `127.0.0.1:5672` 可访问
5. 检查 `FriendMatch/src/main/resources/application.yaml` 中配置项是否与本机一致
6. 启动后端，确认服务运行在 `http://localhost:8081`

### 35.2 前端环境准备

1. 安装 Node.js，版本满足 `^20.19.0 || >=22.12.0`
2. 在 `VueFormatCode` 目录执行 `npm install`
3. 执行 `npm run dev`
4. 确认前端运行地址为 `http://localhost:5173`
5. 确认代理 `/api` 已转发到 `http://localhost:8081`

### 35.3 配置文件检查项

- MySQL：库名、用户名、密码、时区
- Redis：host、port、password、database
- RabbitMQ：host、port、username、password
- 邮件：QQ SMTP 配置是否有效
- OSS：`bucket-name`、`url-prefix`、`upload-url` 是否可用

### 35.4 启动后检查

- 后端接口可访问
- 前端可正常打开登录页
- 登录验证码可正常获取
- 数据库、Redis、RabbitMQ 连接无报错
- 上传接口能成功拿到预签名
