# WebSocket 联调文档

## 一、用途

WebSocket 主要用于：
- 新消息实时推送
- 已读状态同步
- 消息编辑通知
- 消息撤回通知
- 系统通知推送
- 群公告变更推送（如有）

## 二、前端连接时机

- 登录成功后建立连接
- 退出登录时主动断开
- token 失效时断开并跳回登录

## 三、前端建议处理的事件类型

### 1. 新消息
- 刷新当前会话消息流
- 非当前会话增加未读数
- 刷新会话列表最后一条消息

### 2. 已读回执
- 更新消息已读状态
- 刷新未读计数

### 3. 消息编辑
- 替换原消息内容
- 显示“已编辑”标记

### 4. 消息撤回
- 将消息替换为“该消息已撤回”或直接移除

### 5. 系统通知
- 增加通知未读数
- 通知中心实时插入新通知

## 四、前端建议的数据结构

```json
{
  "event": "message_new",
  "data": {}
}
```

推荐事件名：
- `message_new`
- `message_read`
- `message_edit`
- `message_revoke`
- `system_notice`
- `group_notice_update`

## 五、联调建议

- HTTP 负责拉历史和执行动作
- WebSocket 负责实时同步
- 前端应支持断线重连
- 重连成功后主动刷新未读数与当前会话
