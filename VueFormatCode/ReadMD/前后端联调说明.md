# 前后端联调说明

## 一、联调基础

前端联调前需要确认：
- 后端已正常启动
- 数据库已导入 `SQL/friendmatch.sql`
- Redis 已启动
- 邮件服务已配置
- 对象存储配置已可用
- WebSocket 可连接

## 二、联调顺序建议

1. 认证模块
2. 用户资料模块
3. 团队模块
4. 聊天模块
5. 聊天增强模块
6. 搜索推荐模块
7. 举报 / 申诉 / 反馈 / 设备管理

## 三、前端联调注意事项

- 所有业务失败统一读 `message`
- 不要在前端硬编码旧角色 `role_type=4`
- 时间字段统一按时间字符串处理
- 列表页要兼容空列表
- 权限按钮显隐以当前用户角色和接口返回为准

## 四、推荐联调环境

- 一个普通用户账号
- 一个管理员账号
- 一个队长账号
- 至少一个团队
- 至少两名可互发消息的用户
