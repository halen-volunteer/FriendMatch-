# FriendMatch 前端接口文档

> 面向对象：前端开发 / 联调人员
> 版本基线：当前 `Part1 ~ Part7` 实现文档
> 重要说明：团队角色已按最新口径收敛为 **1-队长、2-管理员、3-普通成员**，**不再包含嘉宾角色**。

---

## 一、统一约定

### 1.1 响应结构

后端统一响应结构：

```json
{
  "success": true,
  "data": {},
  "message": null
}
```

失败示例：

```json
{
  "success": false,
  "data": null,
  "message": "参数不合法"
}
```

### 1.2 鉴权方式

- 登录成功后前端保存 token
- 后续请求统一携带认证头
- `/api/auth/captcha`、`/api/auth/code`、`/api/auth/register`、`/api/auth/login` 为匿名接口
- 其余接口默认需要登录

### 1.3 分页约定

常见分页参数：

- `page`：页码，默认 1
- `pageSize`：每页条数，默认 20

常见分页返回：

```json
{
  "success": true,
  "data": [],
  "total": 0,
  "message": null
}
```

## 二、Part1 用户认证

### 2.1 获取图形验证码

- `GET /api/auth/captcha`
- 是否登录：否
- 返回：响应头 `captchaId`，响应体为验证码图片二进制

前端处理：
- 直接把接口返回作为图片展示
- 读取响应头中的 `captchaId`
- 登录时一并提交 `captchaId + captchaCode`

### 2.2 发送邮箱验证码

- `POST /api/auth/code`
- 是否登录：否

请求体：

```json
{
  "email": "user@example.com"
}
```

成功返回：

```json
{
  "success": true,
  "data": "验证码发送成功",
  "message": null
}
```

### 2.3 注册

- `POST /api/auth/register`
- 是否登录：否

请求体：

```json
{
  "userNickname": "张三",
  "userEmail": "zhangsan@example.com",
  "userPassword": "Aa123456",
  "emailCode": "123456"
}
```

成功返回核心字段：

```json
{
  "success": true,
  "data": {
    "id": 1001,
    "userAccount": "1234567890",
    "userNickname": "张三",
    "userAvatar": "",
    "userEmail": "zhangsan@example.com",
    "userTags": "",
    "userIntro": ""
  },
  "message": null
}
```

### 2.4 登录

- `POST /api/auth/login`
- 是否登录：否

建议前端支持两种登录方式：
- 账号密码登录
- 验证码登录（若后端已开放）

前端应兼容以下登录结果：
- 返回 token
- 返回当前用户基础信息
- 返回处罚/封禁类提示信息

### 2.5 获取当前登录用户

- `GET /api/auth/me`
- 是否登录：是

前端用途：
- 页面初始化恢复登录态
- 顶部用户信息展示
- 权限初始化

## 三、Part2 用户管理

### 3.1 编辑个人资料

- `POST /api/user/profile/update`
- 是否登录：是

请求体：

```json
{
  "userNickname": "新昵称",
  "userIntro": "新的个人简介",
  "userAvatar": "https://example.com/avatar.jpg",
  "userTags": ["Java", "后端"]
}
```

### 3.2 查看用户资料

- `GET /api/user/{userId}/profile`
- 是否登录：是

返回核心字段：

```json
{
  "success": true,
  "data": {
    "id": 1002,
    "userAccount": "9876543210",
    "userNickname": "李四",
    "userAvatar": "https://example.com/avatar/1002.jpg",
    "userTags": "游戏,音乐",
    "userIntro": "大家好",
    "friendStatus": 1
  },
  "message": null
}
```

### 3.3 获取隐私设置

- `GET /api/user/privacy`

### 3.4 更新隐私设置

- `POST /api/user/privacy/update`

请求体：

```json
{
  "viewInfo": 1,
  "sendMsg": 1,
  "searchByEmail": 0
}
```

### 3.5 好友关系相关

前端需准备以下能力页：
- 发送好友申请
- 同意好友申请
- 拒绝好友申请
- 删除好友
- 获取好友列表
- 获取好友申请列表

### 3.6 黑名单相关

前端需准备以下能力页：
- 加入黑名单
- 移出黑名单
- 获取黑名单列表

## 四、Part3 团队管理

### 4.1 创建团队

- `POST /api/team/create`

请求体：

```json
{
  "teamName": "Java学习小组",
  "teamDesc": "一起学习 Java",
  "teamTags": ["Java", "后端"],
  "teamType": 1,
  "joinRule": 1,
  "joinPassword": "",
  "maxMember": 200
}
```

### 4.2 团队列表

- `GET /api/team/list`

查询参数：
- `teamType`
- `sortBy`
- `page`
- `pageSize`

### 4.3 搜索团队

- `GET /api/team/search`

查询参数：
- `type=name|tag`
- `keyword`
- `page`
- `pageSize`

### 4.4 团队详情

- `GET /api/team/{teamId}`

### 4.5 更新团队

- `POST /api/team/update`

### 4.6 解散团队

- `POST /api/team/dissolve`

### 4.7 申请入队

- `POST /api/team/apply`

### 4.8 审核入队申请

- `POST /api/team/apply/audit`

### 4.9 团队成员相关

前端需支持：
- 成员列表
- 移除成员
- 团队内禁言/解除禁言
- 转移队长
- 设置管理员/取消管理员
- 退出团队

### 5.1 发送私聊消息

- `POST /api/chat/private/send`

请求体示例：

```json
{
  "recvUserId": 1002,
  "msgType": 1,
  "msgContent": "你好"
}
```

- `GET /api/chat/private/history`

- `POST /api/chat/team/send`

请求体示例：

```json
{
  "teamId": 1001,
  "msgType": 1,
  "msgContent": "大家好",
  "atUserIds": [1002, 1003]
}
```

### 5.4 查询群聊历史

- `GET /api/chat/team/history`

### 5.5 标记已读

- `POST /api/chat/read`

### 5.6 获取未读会话统计

- `GET /api/chat/unread`

前端展示建议：
- 总未读数
- 会话未读数
- 最后一条消息摘要
- 时间戳

## 七、Part6 聊天增强与消息治理

- `POST /api/oss/presign`

前端流程：
1. 先申请上传凭证
2. 拿到上传地址与 `fileUrl`
3. 前端直传对象存储

## 九、前端联调重点提醒

1. **角色字段已无嘉宾**：前端不要再写 `role_type=4` 的展示与权限判断。
2. **聊天类型**：`msgType` 需要兼容文本、图片、文件、表情包、@消息。
3. **聊天实时能力**：HTTP 负责拉历史和提交动作，WebSocket 负责实时推送。
4. **搜索/推荐**：结果具备缓存，不保证每次请求都即时重算。
5. **处罚/禁言**：前端发送失败时应优先展示后端返回的业务提示，不要自行猜测原因。

1. 登录 / 注册 / 获取当前用户
2. 个人中心 / 用户资料 / 隐私设置
3. 团队列表 / 团队详情 / 创建团队
4. 私聊 / 群聊基础页
5. 未读数 / 系统通知
6. 消息增强（编辑、撤回、收藏、置顶、搜索）
7. 搜索 / 推荐
8. 举报 / 申诉 / 反馈 / 设备管理
