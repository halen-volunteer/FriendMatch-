# 前端接口文档_详细版

> 目录：`VueFormatCode/ReadMD`
> 用途：前端可直接按此文档开始开发页面、状态和联调逻辑。
> 说明：本文档基于当前后端实现文档整理，部分接口路径若后端最终代码命名略有差异，以实际 Controller 为准，但页面开发、类型设计、状态流转可直接启动。
> 重要：团队角色已取消嘉宾，仅保留 `1-队长 / 2-管理员 / 3-普通成员`。

---

## 一、统一返回结构

### 成功

```json
{
  "success": true,
  "data": {},
  "message": null
}
```

### 失败

```json
{
  "success": false,
  "data": null,
  "message": "参数不合法"
}
```

前端统一处理规则：
- `success=true`：正常进入业务流
- `success=false`：统一 toast / message 提示 `message`
- `401/未登录类语义`：跳转登录
- `无权限`：禁用按钮 + 提示

## 二、认证模块

## 2.1 获取图形验证码

- 接口：`GET /api/auth/captcha`
- 登录：否
- 请求参数：无
- 响应：图片二进制 + 响应头 `captchaId`

前端要做：
- 用 `<img>` 或 blob URL 展示验证码
- 保存 `captchaId`
- 支持点击刷新

## 2.2 发送邮箱验证码

- 接口：`POST /api/auth/code`
- 登录：否

请求体：

```json
{
  "email": "user@example.com"
}
```

前端要做：
- 按钮发送后进入 60 秒倒计时
- 后端失败时展示 message

## 2.3 注册

- 接口：`POST /api/auth/register`
- 登录：否

请求体：

```json
{
  "userNickname": "张三",
  "userEmail": "zhangsan@example.com",
  "userPassword": "Aa123456",
  "emailCode": "123456"
}
```

成功返回建议前端类型：

```ts
interface UserFormat {
  id: number
  userAccount: string
  userNickname: string
  userAvatar: string
  userEmail: string
  userTags: string
  userIntro: string
}
```

## 2.4 登录

- 接口：`POST /api/auth/login`
- 登录：否

建议前端按以下结构预留：

```ts
interface LoginResult {
  token: string
  userInfo: UserFormat
}
```

前端要做：
- 保存 token
- 拉取 `/api/auth/me`
- 建立 WebSocket
- 跳转首页

## 2.5 获取当前登录用户

- 接口：`GET /api/auth/me`
- 登录：是

前端用途：
- 应用启动恢复登录态
- 顶栏用户信息显示
- 判断角色、权限、菜单

## 三、用户模块

## 3.1 编辑个人资料

- 接口：`POST /api/user/profile/update`

请求体：

```json
{
  "userNickname": "新昵称",
  "userIntro": "新的个人简介",
  "userAvatar": "https://example.com/avatar.jpg",
  "userTags": ["Java", "后端"]
}
```

前端表单规则：
- 昵称：3-16 位
- 简介：<=512
- 标签：最多 5 个，每个 <=20 字符

## 3.2 查看用户资料

- 接口：`GET /api/user/{userId}/profile`

返回建议类型：

```ts
interface UserProfileVO {
  id: number
  userAccount: string
  userNickname: string
  userAvatar: string
  userTags: string
  userIntro: string
  friendStatus?: number
}
```

## 3.3 获取隐私设置

- 接口：`GET /api/user/privacy`

返回：

```json
{
  "viewInfo": 1,
  "sendMsg": 1,
  "searchByEmail": 0
}
```

## 3.4 更新隐私设置

- 接口：`POST /api/user/privacy/update`

请求体：

```json
{
  "viewInfo": 1,
  "sendMsg": 1,
  "searchByEmail": 0
}
```

前端要做：
- 单选或分段选择器
- 保存后提示“设置已更新”

## 3.5 好友与黑名单

前端建议拆出以下 API 服务层函数：

```ts
applyFriend(userId: number)
approveFriend(applyId: number)
rejectFriend(applyId: number)
removeFriend(friendId: number)
getFriendList()
getFriendApplyList()
addBlacklist(userId: number)
removeBlacklist(userId: number)
getBlacklistList()
```

## 四、团队模块

## 4.1 创建团队

- 接口：`POST /api/team/create`

请求体：

```json
{
  "teamName": "Java学习小组",
  "teamDesc": "一起学习 Java",
  "teamTags": ["Java", "后端"],
  "teamType": 1,
  "joinRule": 1,
  "joinPassword": "",
  "maxMember": 200
}
```

返回：团队 ID 或成功标记。

## 4.2 团队列表

- 接口：`GET /api/team/list`

查询参数：
- `teamType?`
- `sortBy?`
- `page`
- `pageSize`

建议返回项前端类型：

```ts
interface TeamCardVO {
  id: number
  teamName: string
  teamAvatar: string
  teamIntro: string
  teamTags: string
  creatorId: number
  maxMember: number
  teamType: number
  joinRule: number
  teamAllMute: number
  memberCount: number
  isMember?: number
}
```

## 4.3 搜索团队

- 接口：`GET /api/team/search`
- 参数：`type=name|tag`、`keyword`、`page`、`pageSize`

## 4.4 团队详情

- 接口：`GET /api/team/{teamId}`

建议前端在详情页准备：
- 团队信息卡
- 成员数
- 我的角色
- 入队按钮 / 退出按钮 / 管理按钮

## 4.5 团队管理动作

前端建议服务层函数：

```ts
updateTeam(data)
dissolveTeam(teamId: number)
applyJoinTeam(data)
auditJoinApply(data)
getTeamMembers(teamId: number)
setTeamAdmin(data)
removeTeamAdmin(data)
removeTeamMember(data)
muteTeamMember(data)
unmuteTeamMember(data)
transferLeader(data)
quitTeam(teamId: number)
```

### 团队角色判断

```ts
type TeamRole = 1 | 2 | 3
```

- `1`：队长
- `2`：管理员
- `3`：普通成员

不要再写 `4`。

## 五、聊天模块

## 5.1 私聊发送

- 接口：`POST /api/chat/private/send`

请求体：

```json
{
  "recvUserId": 1002,
  "msgType": 1,
  "msgContent": "你好"
}
```

建议前端消息类型：

```ts
type MsgType = 1 | 2 | 3 | 4 | 5
```

## 5.2 群聊发送

- 接口：`POST /api/chat/team/send`

请求体：

```json
{
  "teamId": 1001,
  "msgType": 1,
  "msgContent": "大家好",
  "atUserIds": [1002, 1003]
}
```

## 5.3 历史消息查询

- `GET /api/chat/private/history`
- `GET /api/chat/team/history`

建议返回结构：

```ts
interface ChatMessageVO {
  id: number
  senderId: number
  recvType: number
  recvId: number
  conversationId: string
  msgType: MsgType
  msgContent: string
  isEdited: number
  createTime: string
}
```

## 5.4 标记已读

- 接口：`POST /api/chat/read`

前端要做：
- 进入会话页时触发
- 滚动到底部或首屏加载完成后触发

## 5.5 未读统计

- 接口：`GET /api/chat/unread`

建议前端会话列表类型：

```ts
interface ConversationUnreadVO {
  conversationId: string
  unreadCount: number
  lastMsg?: string
  lastTime?: string
}
```
