## 二、通用字段说明

为避免各接口重复解释，统一说明高频字段：

- `userId`：用户 ID
- `friendId`：好友用户 ID / 对方用户 ID
- `teamId`：团队 ID
- `msgId` / `messageId`：消息 ID
- `reportId`：举报记录 ID
- `pinId`：置顶记录 ID
- `collectionId`：收藏记录 ID
- `conversationId`：会话 ID，私聊如 `1_2`，群聊如 `team_1001`
- `page`：页码
- `pageSize`：每页数量
- `keyword`：搜索关键词
- `deviceId`：设备唯一标识
- `fileName`：文件名
- `fileSize`：文件大小，单位字节
- `fileUrl`：文件访问地址
- `uploadUrl`：上传地址
- `uploadToken`：上传凭证
- `captchaId`：图形验证码标识
- `token`：登录令牌

### 通用返回结构说明

除验证码图片流等特殊接口外，后端统一返回 `Result`：

- `success`：是否成功，`true-成功，false-失败`
- `errorMsg`：错误信息，失败时返回
- `data`：业务数据主体
- `total`：总条数，分页接口返回

分页类接口通常为：
- `success`：是否成功
- `data`：当前页数据列表
- `total`：总记录数

非分页类接口通常为：
- `success`：是否成功
- `data`：单对象、字符串提示或 `null`
- `errorMsg`：失败原因

## 三、用户认证与登录模块

### 1. `LoginController`
基础路径：`/api/auth`

#### 已实现接口
- `GET /api/auth/captcha` 【前端已完成】
  - 功能：生成图形验证码
  - 入参：无
  - 返回：响应头 `captchaId`（图形验证码标识），响应体为验证码图片流

- `POST /api/auth/register` 【前端已完成】
  - 功能：用户注册
  - 入参：`username`（用户名）、`email`（注册邮箱）、`emailCode`（邮箱验证码）、`password`（登录密码）
  - 返回：`success`、`errorMsg`、`data`
  - `data`：通常为注册成功提示或为空

- `POST /api/auth/code` 【前端已完成】

  - 入参：`email`（目标邮箱地址）
  - 返回：`success`、`errorMsg`、`data`
  - `data`：通常为发送结果提示

- `POST /api/auth/me` 【前端已完成】
  - 功能：获取当前登录用户信息
