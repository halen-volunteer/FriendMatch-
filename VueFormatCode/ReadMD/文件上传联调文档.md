# 文件上传联调文档

## 一、适用范围

适用于图片消息、文件消息等上传场景。

## 二、标准流程

1. 前端调用 `POST /api/oss/presign`
2. 后端校验文件类型和大小
3. 后端返回上传地址和 `fileUrl`
4. 前端直传对象存储
5. 上传成功后，在聊天发送接口中把 `fileUrl` 作为 `msgContent` 提交

## 三、前端注意事项

- 上传前先校验文件大小和类型
- 上传失败要允许重试
- 上传成功但消息发送失败时，前端应提示“文件已上传，但消息未发送成功”
- 图片与普通文件应使用不同 UI 展示

## 四、消息发送约定

### 图片消息
- `msgType = 2`
- `msgContent = fileUrl`

### 文件消息
- `msgType = 3`
- `msgContent = fileUrl`
