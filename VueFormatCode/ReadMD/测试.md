# FriendMatch 项目测试方案

## 1. 文档目标

本文档用于指导 `VueFormatCode` 前端与 `FriendMatch` 后端的联调测试、功能测试、异常测试、回归测试与上线验收。

本方案的目标不是只验证“接口是否返回成功”，而是同时保证：
- 功能正确
- 页面与接口联动正确
- 权限与异常处理正确
- 数据状态一致
- 具备回归与上线验收依据

---

## 6. 执行原则

### 6.1 执行顺序
1. 环境检查
2. 登录与鉴权
3. 用户资料 / 隐私
4. 好友 / 黑名单
5. 团队
6. 聊天
7. 搜索与推荐
8. 通知 / 设备 / 在线状态
9. 举报 / 申诉 / 反馈 / 处罚
10. 上传
11. 回归
12. 性能与安全基础项

### 6.2 优先级定义
- `P0`：主链路阻塞功能，每轮必测
- `P1`：核心高频功能，每轮必测
- `P2`：重要但非高频，版本回归测
- `P3`：低频或兼容项，按需测

### 6.3 用例通过标准
同时满足以下条件才算通过：
- HTTP 状态和业务返回正确
- 页面展示与交互符合预期
- 数据、缓存、列表状态一致
- 无前端控制台报错与后端异常日志

## 7. 通用检查清单

### 7.1 所有接口都检查
- 方法、URL、Header、Query、Body 是否正确
- `Authorization` 是否按预期携带
- 响应结构是否符合项目约定：`success`、`errorMsg/message`、`data`、`total`

### 7.2 关键接口必须补测异常
- 未登录访问
- token 无效/过期
- 缺少必填参数
- 参数类型错误
- 枚举值非法
- 资源不存在
- 权限不足
- 重复提交 / 重复处理

### 7.3 页面联调必须补查
- 按钮和表单是否可触发
- 成功 / 失败提示是否清晰
- 列表是否刷新
- 页面状态是否同步
- 路由跳转是否正确

## 8. 模块级测试重点

### 8.1 登录认证
关键链路：验证码 -> 登录 -> 获取当前用户；注册 -> 邮箱验证码 -> 注册；忘记密码 -> 新密码登录。
必测异常：错误密码、错误验证码、邮箱已注册、验证码过期、未登录访问 `/api/auth/me`。

### 8.2 用户资料与隐私
关键链路：修改头像/昵称/简介 -> 重新获取资料 -> 页面同步；修改隐私设置 -> 其他账号访问验证。
必测异常：昵称超长、tags 非法、访问不存在用户、未登录修改资料。

### 8.3 好友关系
关键链路：发起申请 -> 查看申请 -> 同意/拒绝 -> 列表更新 -> 删除好友。
必测异常：不能加自己、重复申请、已是好友重复申请、非接收方审批。

### 8.4 黑名单
关键链路：拉黑 -> 验证好友/私聊受影响 -> 解除拉黑 -> 能力恢复。
必测异常：重复拉黑、解除未拉黑对象、拉黑不存在用户。

### 8.5 团队
关键链路：创建团队 -> 申请/密码加入 -> 审批 -> 成员列表 -> 修改角色 -> 转让队长 -> 全员禁言/成员禁言 -> 退出/解散。
必测异常：密码错误、满员加入、普通成员审批失败、普通成员禁言失败、已解散团队继续操作。

### 8.6 聊天
关键链路：
- 私聊：发送 -> 历史 -> 编辑 -> 撤回 -> 收藏 -> 置顶 -> 搜索 -> 已读
- 群聊：发送 -> 历史 -> 公告 -> 群禁言
- 举报：举报 -> 查状态 -> 管理员查看上下文 -> 管理员处理 -> 发起申诉
必测异常：空消息、非发送者编辑/撤回、非成员发群消息、举报不存在消息、非管理员访问管理接口、已禁言成员仍可发言。

### 8.7 OSS 上传
关键链路：获取预签名 -> 实际上传 -> URL 可访问 -> 头像/消息中可用。
必测异常：文件为空、类型非法、文件过大、token 过期。

### 8.8 搜索与推荐
关键链路：搜索 -> 历史 -> 热搜 -> 联想 -> 推荐列表 -> 点击推荐上报。
必测异常：空关键词、超长关键词、非法 `type/searchType`、推荐点击参数非法。

### 8.9 通知 / 设备 / 在线状态
- 通知：业务动作触发通知 -> 未读数变化 -> 列表 -> 已读 -> 删除
- 设备：绑定 -> 列表 -> 信任 -> 下线 -> 删除
- 在线状态：设置 -> 心跳 -> 查询 -> 下线

### 8.10 举报 / 申诉 / 反馈 / 处罚
要求覆盖完整业务闭环：提交 -> 查询/列表 -> 管理员处理 -> 用户查看结果/恢复状态。
必测异常：越权处理、重复处理、资源不存在。

## 9. 105 个接口最低执行要求

> 每个接口至少 1 条正常流；关键接口再补异常与权限用例。

- FriendManagementController：发起申请、好友列表、同意、拒绝、删除、申请列表
- BlacklistManagementController：拉黑、解除拉黑、黑名单列表
- TeamController：创建、列表、搜索、详情、更新、解散、申请加入、密码加入、邀请、退出、待审批、审批、成员列表、按角色筛选、移除成员、角色更新、兼容角色接口、转让队长、兼容转让接口、全员禁言、成员禁言、成员解除禁言
- ChatController：私聊发送、私聊历史、群聊发送、群聊历史、编辑、撤回、收藏、取消收藏、收藏列表、置顶、取消置顶、置顶列表、搜索、消息举报、举报状态、管理员上下文、管理员列表、管理员处理、举报申诉、已读、未读数、设置公告、获取公告、兼容全员禁言、兼容成员禁言、兼容解除禁言、已取消群名接口返回失败提示
- OssController：预签名获取且可用于上传
- SearchRecommendController：搜索用户、搜索团队、搜索历史、清空历史、热搜、联想、推荐用户、推荐团队、推荐点击
- SystemNoticeController：未读数、通知列表、标已读、删除通知
- DeviceController：绑定设备、设备列表、信任、下线、删除
- OnlineStatusController：设置状态、心跳、查询状态、主动下线
- ReportController：举报用户、举报团队、用户举报状态、团队举报状态、统一状态查询、我的举报列表、用户举报管理列表、处理用户举报、团队举报管理列表、处理团队举报
- AppealController：提交申诉、我的申诉、待处理列表、处理申诉
- FeedbackController：提交反馈、我的反馈列表、反馈详情、处理反馈、反馈管理列表
- PunishController：执行处罚、撤销处罚、处罚记录、我的处罚记录、违规统计

## 15. 上线验收标准

满足以下条件方可建议上线：
- P0 用例全部通过

- 高缺陷 = 0

## 16. 结论

当前旧版测试文档的优点是“接口覆盖广”，但不足是“执行颗粒度不足、环境信息过时、异常/权限/联调/回归不完整”。

本方案优化后，已经补齐：
- 当前真实环境基线
- 测试账号与前置数据要求
- 模块级测试重点
- 异常、权限、联调、回归要求
- 性能与安全基础项
- 执行模板与上线验收口径

执行测试时，必须以“业务闭环 + 页面联动 + 状态一致性”作为通过标准，而不是只看接口是否返回成功。
