## 一、项目概述

FriendMatch 是一款面向兴趣社交的 Web 应用，支持用户注册登录、好友管理、团队创建与加入、实时聊天等核心功能。前端工程位于 `VueFormatCode/` 目录下，作为独立 SPA（单页应用）与后端 RESTful API 及 WebSocket 服务通信。

### 核心功能模块

| 模块 | 功能描述 | 优先级 |
|------|----------|--------|
| 用户认证 | 注册、登录、找回密码、图形验证码 | P0 |
| 用户管理 | 个人资料、隐私设置、好友关系、黑名单 | P0 |
| 团队管理 | 创建团队、搜索加入、审批、成员管理、禁言 | P0 |
| 聊天系统 | 私聊、群聊、消息撤回、已读回执、未读统计 | P0 |
| 系统通知 | 好友申请、审批结果、处罚通知、反馈回复 | P1 |
| 处罚与反馈 | 查看处罚记录、提交申诉与反馈 | P1 |
| 在线状态 | 在线/离开/忙碌/隐身、心跳保活 | P1 |

### 6.1 当前配置

`src/router/index.js` 使用 `createWebHistory` 模式（基于 HTML5 History API），路由表当前为空，需按下方规划填充。

### 6.2 路由表规划

```js
// src/router/index.js
import { createRouter, createWebHistory } from 'vue-router'
import { useAuthStore } from '@/stores/auth'

const router = createRouter({
  history: createWebHistory(import.meta.env.BASE_URL),
  routes: [
    // 认证页面（无需登录）
    {
      path: '/login',
      name: 'Login',
      component: () => import('@/views/auth/LoginView.vue'),
      meta: { requiresAuth: false },
    },
    {
      path: '/register',
      name: 'Register',
      component: () => import('@/views/auth/RegisterView.vue'),
      meta: { requiresAuth: false },
    },
    {
      path: '/forget-password',
      name: 'ForgetPassword',
      component: () => import('@/views/auth/ForgetPasswordView.vue'),
      meta: { requiresAuth: false },
    },

    // 主布局（需要登录）
    {
      path: '/',
      component: () => import('@/components/layout/MainLayout.vue'),
      meta: { requiresAuth: true },
      children: [
        { path: '', redirect: '/chat' },
        // 聊天
        { path: 'chat', name: 'Chat', component: () => import('@/views/chat/ChatView.vue') },
        { path: 'chat/private/:friendId', name: 'PrivateChat', component: () => import('@/views/chat/PrivateChatView.vue') },
        { path: 'chat/team/:teamId', name: 'TeamChat', component: () => import('@/views/chat/TeamChatView.vue') },
        // 用户
        { path: 'profile', name: 'MyProfile', component: () => import('@/views/user/MyProfileView.vue') },
        { path: 'profile/:userId', name: 'UserProfile', component: () => import('@/views/user/UserProfileView.vue') },
        { path: 'friends', name: 'Friends', component: () => import('@/views/user/FriendsView.vue') },
        { path: 'blacklist', name: 'Blacklist', component: () => import('@/views/user/BlacklistView.vue') },
        // 团队
        { path: 'teams', name: 'Teams', component: () => import('@/views/team/TeamListView.vue') },
        { path: 'teams/:teamId', name: 'TeamDetail', component: () => import('@/views/team/TeamDetailView.vue') },
        { path: 'teams/:teamId/members', name: 'TeamMembers', component: () => import('@/views/team/TeamMembersView.vue') },
        // 通知与反馈
        { path: 'notices', name: 'Notices', component: () => import('@/views/user/NoticesView.vue') },
        { path: 'feedback', name: 'Feedback', component: () => import('@/views/user/FeedbackView.vue') },
      ],
    },

    // 兜底重定向
    { path: '/:pathMatch(.*)*', redirect: '/' },
  ],
})

// 全局路由守卫
router.beforeEach((to) => {
  const authStore = useAuthStore()
  if (to.meta.requiresAuth && !authStore.isLoggedIn) {
    return { name: 'Login' }
  }
  if (!to.meta.requiresAuth && authStore.isLoggedIn) {
    return { name: 'Chat' }
  }
})

export default router
```

### 6.3 路由守卫说明

| 场景 | 处理方式 |
|------|----------|
| 访问需登录页面但未登录 | 重定向至 `/login` |
| 已登录访问 `/login`、`/register` | 重定向至 `/chat` |
| API 返回 401（Token 过期） | 拦截器清除状态 → 跳转 `/login` |

## 七、状态管理

使用 Pinia 进行全局状态管理，所有 Store 采用 **Composition API 风格**（与现有 `counter.js` 一致）。

### 7.1 认证 Store（src/stores/auth.js）

```js
import { ref, computed } from 'vue'
import { defineStore } from 'pinia'

export const useAuthStore = defineStore('auth', () => {
  // 从 localStorage 恢复登录状态
  const token = ref(localStorage.getItem('token') || '')
  const userInfo = ref(JSON.parse(localStorage.getItem('userInfo') || 'null'))

  const isLoggedIn = computed(() => !!token.value)
  const userId = computed(() => userInfo.value?.userId)
  const userNickname = computed(() => userInfo.value?.userNickname)
  const userAvatar = computed(() => userInfo.value?.userAvatar)

  function setAuth(data) {
    token.value = data.token
    userInfo.value = {
      userId: data.userId,
      userNickname: data.userNickname,
      userAvatar: data.userAvatar,
    }
    localStorage.setItem('token', data.token)
    localStorage.setItem('userInfo', JSON.stringify(userInfo.value))
  }

  function clearAuth() {
    token.value = ''
    userInfo.value = null
    localStorage.removeItem('token')
    localStorage.removeItem('userInfo')
  }

  return { token, userInfo, isLoggedIn, userId, userNickname, userAvatar, setAuth, clearAuth }
})
```

### 7.2 聊天 Store（src/stores/chat.js）

```js
import { ref } from 'vue'
import { defineStore } from 'pinia'

export const useChatStore = defineStore('chat', () => {
  // key: conversationId, value: 未读数
  const unreadMap = ref({})
  // 当前激活的会话ID
  const activeConversationId = ref('')
  // 会话列表（最近联系人）
  const conversationList = ref([])

  const totalUnread = () =>
    Object.values(unreadMap.value).reduce((sum, n) => sum + n, 0)

  function setUnread(conversationId, count) {
    unreadMap.value[conversationId] = count
  }

  function clearUnread(conversationId) {
    unreadMap.value[conversationId] = 0
  }

  function setActiveConversation(id) {
    activeConversationId.value = id
  }

  return { unreadMap, activeConversationId, conversationList, totalUnread, setUnread, clearUnread, setActiveConversation }
})
```

### 7.3 通知 Store（src/stores/notice.js）

```js
import { ref } from 'vue'
import { defineStore } from 'pinia'

export const useNoticeStore = defineStore('notice', () => {
  const unreadCount = ref(0)

  function setUnreadCount(count) {
    unreadCount.value = count
  }

  function decrement(n = 1) {
    unreadCount.value = Math.max(0, unreadCount.value - n)
  }

  return { unreadCount, setUnreadCount, decrement }
})
```

## 八、API 接口封装

### 8.1 Axios 实例（src/api/request.js）

需先安装 Axios：

```sh
npm install axios
```

```js
import axios from 'axios'
import { useAuthStore } from '@/stores/auth'
import router from '@/router'

const request = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || 'http://localhost:8080',
  timeout: 10000,
})

// 请求拦截器：自动附加 Token
request.interceptors.request.use((config) => {
  const authStore = useAuthStore()
  if (authStore.token) {
    config.headers['Authorization'] = authStore.token
  }
  return config
})

// 响应拦截器：统一处理 401
request.interceptors.response.use(
  (response) => response.data,
  (error) => {
    if (error.response?.status === 401) {
      const authStore = useAuthStore()
      authStore.clearAuth()
      router.push({ name: 'Login' })
    }
    return Promise.reject(error)
  }
)

export default request
```

## 九、WebSocket 集成

### 9.1 连接规范

- 连接地址：`ws://{host}/ws?token={token}`
- Token 通过 URL 参数传递，服务端在握手时校验
- 同一用户新连接建立时，旧连接自动断开
- 用户上线后自动补推离线期间未读消息（每会话最多 20 条）

### 9.2 WebSocket 管理类（src/utils/websocket.js）

```js
import { useAuthStore } from '@/stores/auth'
import { useChatStore } from '@/stores/chat'

const WS_BASE_URL = import.meta.env.VITE_WS_BASE_URL || 'ws://localhost:8080'

let ws = null
let heartbeatTimer = null
let reconnectTimer = null
let reconnectCount = 0
const MAX_RECONNECT = 5

export function connectWebSocket(onMessage) {
  const authStore = useAuthStore()
  if (!authStore.token) return

  ws = new WebSocket(`${WS_BASE_URL}/ws?token=${authStore.token}`)

  ws.onopen = () => {
    console.log('[WS] 连接成功')
    reconnectCount = 0
    startHeartbeat()
  }

  ws.onmessage = (event) => {
    const msg = JSON.parse(event.data)
    onMessage(msg)
  }

  ws.onclose = () => {
    console.log('[WS] 连接断开')
    stopHeartbeat()
    scheduleReconnect(onMessage)
  }

  ws.onerror = (err) => {
    console.error('[WS] 错误', err)
  }
}

export function disconnectWebSocket() {
  stopHeartbeat()
  clearTimeout(reconnectTimer)
  if (ws) {
    ws.close()
    ws = null
  }
}

function startHeartbeat() {
  heartbeatTimer = setInterval(() => {
    if (ws?.readyState === WebSocket.OPEN) {
      ws.send(JSON.stringify({ type: 'heartbeat' }))
    }
  }, 30000) // 每 30 秒发送一次心跳
}

function stopHeartbeat() {
  clearInterval(heartbeatTimer)
}

function scheduleReconnect(onMessage) {
  if (reconnectCount >= MAX_RECONNECT) return
  reconnectCount++
  reconnectTimer = setTimeout(() => {
    console.log(`[WS] 第 ${reconnectCount} 次重连...`)
    connectWebSocket(onMessage)
  }, 3000 * reconnectCount)
}
```

### 9.3 服务端推送消息类型

| type | 说明 | 关键字段 |
|------|------|----------|
| `private_message` | 私聊消息 | msgId, senderId, msgType, content, createTime, offline |
| `team_message` | 群聊消息 | msgId, teamId, senderId, msgType, content, createTime, offline |
| `message_revoke` | 消息撤回通知 | msgId, conversationId |

### 9.4 消息处理建议

```js
// 在主布局组件 onMounted 中调用
import { connectWebSocket } from '@/utils/websocket'
import { useChatStore } from '@/stores/chat'
import { saveMessageToLocal } from '@/utils/localDb'

const chatStore = useChatStore()

connectWebSocket((msg) => {
  switch (msg.type) {
    case 'private_message':
    case 'team_message':
      // 1. 写入本地 IndexedDB
      saveMessageToLocal(msg.data)
      // 2. 更新未读数
      const convId = msg.type === 'private_message'
        ? buildConversationId(msg.data.senderId)
        : `team_${msg.data.teamId}`
      chatStore.setUnread(convId, (chatStore.unreadMap[convId] || 0) + 1)
      break
    case 'message_revoke':
      // 更新本地消息状态为已撤回
      break
  }
})
```

### 9.5 会话 ID 规则

| 类型 | 格式 | 示例 |
|------|------|------|
| 私聊 | `min(uid1,uid2)_max(uid1,uid2)` | `1001_1002` |
| 群聊 | `team_{teamId}` | `team_1001` |

// 构建私聊会话ID
export function buildConversationId(myId, friendId) {
  return [myId, friendId].sort((a, b) => a - b).join('_')
}

## 十二、本地存储方案

参考 FriendMatch V2.0 设计哲学：消息持久化在客户端本地，服务端仅做72小时中转缓冲。

### 12.1 IndexedDB 封装（src/utils/localDb.js）

```js
const DB_NAME = 'friendmatch_chat'
const DB_VERSION = 1
const STORE_NAME = 'messages'

let db = null

export function openDb() {
  return new Promise((resolve, reject) => {
    if (db) return resolve(db)
    const req = indexedDB.open(DB_NAME, DB_VERSION)
    req.onupgradeneeded = (e) => {
      const store = e.target.result.createObjectStore(STORE_NAME, { keyPath: 'msgId' })
      store.createIndex('by_conversation', 'conversationId', { unique: false })
    }
    req.onsuccess = (e) => { db = e.target.result; resolve(db) }
    req.onerror = (e) => reject(e)
  })
}

export async function saveMessageToLocal(msg) {
  const db = await openDb()
  return new Promise((resolve, reject) => {
    const tx = db.transaction(STORE_NAME, 'readwrite')
    tx.objectStore(STORE_NAME).put(msg)
    tx.oncomplete = resolve
    tx.onerror = reject
  })
}

export async function getLocalMessages(conversationId, limit = 20) {
  const db = await openDb()
  return new Promise((resolve, reject) => {
    const tx = db.transaction(STORE_NAME, 'readonly')
    const index = tx.objectStore(STORE_NAME).index('by_conversation')
    const req = index.getAll(IDBKeyRange.only(conversationId))
    req.onsuccess = (e) => {
      // 按时间倒序，取最新 limit 条
      const all = e.target.result.sort((a, b) => b.createTime.localeCompare(a.createTime))
      resolve(all.slice(0, limit))
    }
    req.onerror = reject
  })
}
```

### 12.2 消息本地状态字段

| 字段 | 说明 |
|------|------|
| `msgId` | 服务端返回的消息ID（主键）|
| `conversationId` | 会话ID |
| `senderId` | 发送者ID |
| `msgType` | 消息类型（1文本/2图片/3文件/4表情包）|
| `msgContent` | 消息内容（本地永久存储）|
| `createTime` | 发送时间 |
| `isRevoke` | 是否已撤回 |
| `localStatus` | 0-发送中，1-已送达，2-发送失败 |

### 12.3 消息发送流程

```
用户点击发送
    ↓
生成临时 msgId，写本地（localStatus=0）
    ↓
调用 API 发送消息
    ↓
成功 → 用服务端 msgId 更新本地（localStatus=1）
失败 → 更新本地（localStatus=2），展示重试按钮
```

### 12.4 消息接收流程

```
WebSocket 推送消息
    ↓
写入本地 IndexedDB（永久存储）
    ↓
更新 UI 消息列表
    ↓
调用 readMsgs API 标记已读
```
